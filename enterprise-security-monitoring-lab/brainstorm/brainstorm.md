# 1. What is Vulnerability Scanning?

A **vulnerability scan** is an automated assessment that identifies known security weaknesses in a system, application, or network **before they are exploited**.

It compares software versions, configurations, open ports, and services against a database of known vulnerabilities (CVEs) and security best practices.

> **Goal:** Identify weaknesses so they can be fixed before an attacker finds them.

### Common Vulnerability Scanners

* Nessus                     * OpenVAS (Greenbone)
* Qualys                     * Rapid7 InsightVM

---

# 2. What is an EDR?

**EDR (Endpoint Detection and Response)** is security software installed directly on endpoints (servers, laptops, workstations) that continuously monitors operating system activity to detect, investigate, and respond to suspicious or malicious behavior.

Unlike a vulnerability scanner, an EDR watches **what is happening in real time**.

> **Goal:** Detect and stop attacks after they begin.

### Common EDR Platforms

* CrowdStrike Falcon
* Microsoft Defender for Endpoint
* SentinelOne

---

# 3. Does Nmap Count as Vulnerability Scanning?    **NO, Not by itself.**

Nmap is primarily a **network discovery and port scanning tool**.

It tells you things like:

* Which hosts are alive
* Which ports are open
* Which services are running
* Operating system detection
* Service/version detection

However:

Nmap has the **NSE (Nmap Scripting Engine)**, which includes scripts capable of checking for certain vulnerabilities.

So the interview-ready answer is:

> "Nmap is primarily a network reconnaissance and port-scanning tool. While its scripting engine can identify some vulnerabilities, dedicated tools like Nessus or Qualys perform comprehensive vulnerability assessments."

# Interview/Documentation Answers for Your Project

## 1. How Kali Performs Penetration Testing

> In this project, the Kali Linux VM acts as the attack machine. Using penetration testing tools, it simulates real-world attacks against the DVWA application hosted on the RHEL server, including brute-force attacks, SQL injection, command injection, and malicious file uploads. These simulated attacks generate logs that are collected by Splunk, allowing detection rules, alerts, and incident investigations to be developed and validated.

---

## 2. Where Vulnerability Scanning Fits

> Vulnerability scanning occurs before penetration testing. A scanner such as Nessus assesses the RHEL server for known vulnerabilities, insecure configurations, outdated software, and exposed services without attempting exploitation. The scan results help identify weaknesses that should be remediated or validated through penetration testing.

---

## 3. Where EDR Fits

> An EDR solution such as CrowdStrike Falcon would be installed on the RHEL server to continuously monitor operating system activity. While Splunk analyzes logs generated by services like Apache, the EDR monitors endpoint behavior such as process execution, file creation, privilege escalation, and suspicious commands. If an attack successfully executes on the server, the EDR detects the malicious behavior and can generate alerts or take response actions, with those alerts often forwarded to Splunk for centralized investigation.
